script supports this via `load_companion_local_csv()`. Without the companion CSV,
`foreigners_share` and `migration_background_share` are NULL (not 0.0) — correctly
documented as NaN, not silently zeroed.

### Task 2 — C3 (issue #22): LOR geometry + spatial join + fact table

**Status: done, reviewer-approved, geo-data-scientist sign-off: PASS.**

Commits: `a0f6b91`, `548c44f`
Sign-off doc: `docs/epic-c/C3-geo-signoff.md`

(a) **C3-geo**: `ingestion/berlin/lor/ingest_lor_geometries.py` fetches both LOR vintages
from GDI Berlin WFS (CC BY 3.0 DE):
- Pre-2021: `https://gdi.berlin.de/services/wfs/lor_2019` · 448 PLRs
- LOR 2021: `https://gdi.berlin.de/services/wfs/lor_2021` · 542 PLRs
Outputs: `data/raw/berlin/lor/pre2021.parquet` + `lor_2021.parquet` (geometry as WKB,
CRS EPSG:25833, plr_id as area_code). Uses `shapely` (LGPL-2.1, approved in ADR-0003
implementation note).

(b) **C3-join**: `transform/models/intermediate/int_osm_poi_plr.sql` — spatial join of
OSM POIs to PLR using correct vintage:
- `snapshot_year ≤ 2020` → pre-2021 geometry (448 PLRs)
- `snapshot_year ≥ 2021` → LOR 2021 geometry (542 PLRs)
- Predicate: `ST_Within(ST_Transform(ST_Point(lon,lat), 'EPSG:4326','EPSG:25833',true), geom)`
- `always_xy=true` forces correct (lon,lat)=(x,y) axis order
- Materialised as table (join is expensive; ~1.44M rows with data)
- NULL rate ~9–11% is structural (water/airports/forests); 12% warn threshold detects
  catastrophic misalignments

(c) **C3-fact**: `transform/models/marts/fct_poi_development.sql` —
`(area_code, area_vintage, poi_category_h, snapshot_year, poi_count)`.

**WARN=1 (known, deferred):** `relationships_int_osm_poi_plr_area_code` fires because
`dim_area` still sources only from the 2018 thesis golden (area codes don't include WFS
PLR codes). Wiring `stg_berlin_lor` into `dim_area` is deferred to a follow-up issue.
Severity: warn (not error). Documented in `stg_berlin_lor.sql`.

**Geo-DS sign-off** (full doc: `docs/epic-c/C3-geo-signoff.md`):
